## 核心链路验证

### 路径 A：管理者设定 → 进货 → 营运

| 步骤 | 状态 | 说明 |
|------|------|------|
| 创建材料 | ✅ | 可创建，有分类/标签 |
| 创建供应商 | ✅ | 可创建 |
| 创建配方 + 添加材料 | ⚠️ | 可创建配方和材料项，但 UI 不完善 |
| 创建菜单商品 + 绑定配方 | ✅ | 可选择配方和分类 |
| 进货入库（创建批次） | ✅ | 选择材料、数量、成本、供应商 |
| POS 点餐 + 提交 | ✅ | 一键提交，自动预扣+拆单+KDS |
| KDS 制作 + 出餐 | ✅ | 显示菜品和备注，完成自动实扣 |

**路径 A 结论：✅ 基本可达**。管理者可以通过 UI 完成从 0 到营运的完整设定。

### 路径 B：POS → 厨房 → 出餐 → 扣料

| 步骤 | 状态 | 说明 |
|------|------|------|
| 选择商品 + 规格 + 备注 | ✅ | POS 完整 |
| 提交订单 | ✅ | 一键提交 |
| 预扣库存（FEFO） | ✅ | 按过期日优先 |
| 创建 KDS 小票 | ✅ | 按工作站拆分 |
| 开始制作 | ✅ | KDS 按钮 |
| 完成出餐 → 实扣 | ✅ | 自动转换预扣为实扣 |
| 订单完成 | ✅ | 所有小票完成后自动标记 |

**路径 B 结论：✅ 完全可达**。核心业务链路已完整跑通。
