## 三、安全審計

### 🟡 P2-1: XSS 風險（已知）

**風險**: `dangerouslySetInnerHTML` 在打印模板中使用

**評估**: 
- 內部系統，風險可控
- 後端組裝 HTML，相對安全
- 建議：未來開放富文本時引入 DOMPurify

**狀態**: 📋 記錄，建議關注

---

### 🟡 P2-2: Mutex 中毒風險（已知）

**風險**: `database.rs` 超過 140 處 `lock().unwrap()`

**緩解**:
- SQLite 本地訪問，Panic 概率低
- 生產環境建議監控

**狀態**: 📋 記錄，建議關注

---

### 🟡 P2-3: 錯誤堆疊吞噬（已知）

**風險**: `catch (e) { toast.error(String(e)) }`

**評估**: 
- 功能正常運作
- 非技術人員可能看到長堆棧

**狀態**: 📋 記錄，未來優化
