
📦 餐饮作业系统（配方驱动 · 本地优先）开发文档

⸻

🧭 一、项目摘要（基于对话精确总结）

🎯 目标

构建一套：

从原材料 → 半成品 → 菜单 → 点单 → 出餐 → 扣料 → 报表 的本地餐饮操作系统

并支持：

* 配方驱动库存
* 半成品管理
* 批次追踪（lot / 过期）
* KDS 出餐流
* 标签打印
* 微信小程序点单（作为入口）

⸻

🧠 核心认知（关键结论）

1️⃣ 你做的不是 POS

而是：

餐饮版「制造 + 库存 + 前台系统」

⸻

2️⃣ 市场现状（调研结论）

能力	是否已有成熟方案
配方 / BOM	✅ Odoo / ERPNext
半成品	✅ ERP类支持
批次管理	✅ ERP类支持
KDS	✅ 餐饮 SaaS
微信小程序	✅ 餐饮 SaaS
全部整合	❌ 几乎没有

👉 缺口：单店级“制造 + 餐饮”融合系统

⸻

3️⃣ 最重要系统原则

❌ 不要直接从菜品扣库存
✅ 必须通过「配方 → 原料」扣料

⸻

4️⃣ 系统本质一句话

配方驱动的库存交易系统 + 出餐执行系统

⸻

⸻

🔁 三、核心业务流程

1️⃣ 进货

采购 → 入库 → 生成批次 → 打标签 → 库存增加

⸻

2️⃣ 半成品

原料 → 制作 → 半成品入库

例：

* 茶叶 → 茶汤
* 猪肉 → 卤味

⸻

3️⃣ 点单

点单 → 订单生成 → 拆单 → 推送KDS → 预扣库存

⸻

4️⃣ 出餐

KDS → 制作 → 完成 → 确认扣料

⸻

5️⃣ 库存

预扣 → 实扣 → 回补（取消单）

⸻

🧩 四、系统模块设计

1️⃣ 基础模块

* 原材料
* 单位（kg/g/ml）
* 供应商
* 菜单分类
* 出餐工作站

⸻

2️⃣ 配方模块（核心）

支持：

* 原料 → 半成品
* 半成品 → 菜单
* 多层嵌套
* 规格（大份/小份）

示例：

麻辣小龙虾（中份）
= 小龙虾 500g
+ 辣椒油 50g
+ 香料包 10g

⸻

3️⃣ 库存模块

支持：

* 实时库存
* 批次库存
* 可用库存
* 预扣库存
* FIFO / FEFO

⸻

4️⃣ 批次管理（重点）

字段：

batch_id
material_id
quantity
cost
expire_date
created_at

⸻

5️⃣ 半成品模块

支持：

原料 → 制作 → 半成品 → 再消耗

⸻

6️⃣ POS模块

功能：

* 点餐
* 外带 / 堂食
* 加料
* 备注
* 结算（初期可无支付）

⸻

7️⃣ KDS模块

功能：

* 分工作站
* 状态流转
* 制作时间统计

状态：

待制作 → 制作中 → 已完成

⸻

8️⃣ 标签打印

应用：

* 原料标签
* 半成品标签
* 出餐贴（杯贴）

⸻

9️⃣ 报表模块

* 销售额
* 毛利
* 原料消耗
* 损耗
* 热销商品

⸻

🔟 小程序模块

功能：

* 扫码点餐
* 菜单浏览
* 下单
* 订单状态

⚠️ 十、关键坑位总结

❌ 错误做法

* 直接从菜品扣库存
* 没有库存流水
* 不做半成品
* 不做批次

⸻

✅ 正确做法

* 配方驱动扣料
* 库存交易记录完整
* 半成品独立管理
* 批次追踪
