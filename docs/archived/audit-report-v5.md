## 本轮完成功能

### ✅ 数据库事务支持（P0 已完成）

| 功能 | 状态 | 说明 |
|------|------|------|
| `submit_order_full` 事务 | ✅ | 预扣库存 + 创建 KDS 小票 + 更新订单状态，原子操作 |
| `reserve_inventory_for_order` 事务 | ✅ | 独立预扣操作也使用事务 |
| 语句生命周期管理 | ✅ | 使用 `drop(stmt)` 确保 borrow checker 通过 |

**事务保证**：如果预扣库存失败，不会创建 KDS 小票；如果创建小票失败，预扣会回滚。

## 核心链路验证

### 路径 A：管理者设定 → 进货 → 营运

| 步骤 | 状态 | 说明 |
|------|------|------|
| 系统启动（自动 Seed） | ✅ | 8 种材料、7 个配方、7 个商品、8 个批次 |
| 创建材料 | ✅ | 可创建，有分类/标签 |
| 创建供应商 | ✅ | 可创建 |
| 创建配方 + 添加材料 | ⚠️ | 可创建配方和材料项，但 UI 不完善 |
| 创建菜单商品 + 绑定配方 | ✅ | 可选择配方和分类 |
| 进货入库（创建批次） | ✅ | 选择材料、数量、成本、供应商 |
| POS 点餐 + 提交 | ✅ | 一键提交，自动预扣+拆单+KDS（事务保证） |
| KDS 制作 + 出餐 | ✅ | 显示菜品和备注，完成自动实扣 |

**路径 A 结论：✅ 完全可达**。系统启动即可使用，完整体验从 0 到营运。

### 路径 B：POS → 厨房 → 出餐 → 扣料

| 步骤 | 状态 | 说明 |
|------|------|------|
| 选择商品 + 规格 + 备注 | ✅ | POS 完整 |
| 提交订单 | ✅ | 一键提交（事务：预扣+拆单+KDS） |
| 预扣库存（FEFO） | ✅ | 按过期日优先，事务保证 |
| 创建 KDS 小票 | ✅ | 按工作站拆分，事务保证 |
| 开始制作 | ✅ | KDS 按钮 |
| 完成出餐 → 实扣 | ✅ | 自动转换预扣为实扣 |
| 订单完成 | ✅ | 所有小票完成后自动标记 |

**路径 B 结论：✅ 完全可达**。核心业务链路已完整跑通，事务保证数据一致性。
