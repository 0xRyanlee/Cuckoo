## 二、User Roadmap 逐路径审计

### 路径 A：管理者设定 → 营运（完整链路验证）

| 步骤 | 功能 | DB | API | UI | 可达？ | 阻断点 |
|------|------|----|-----|----|--------|--------|
| A1 | 建立原材料 | ✅ | ✅ | ✅ | ⚠️ 部分 | 创建时无法选分类/标签；无编辑/删除 |
| A2 | 建立供应商 | ✅ | ⚠️ | ✅ | ⚠️ 部分 | 无更新/删除 API |
| A3 | 设定单位与换算 | ✅ | ✅ | ❌ | ❌ | 无单位管理 UI，仅预置 seed |
| A4 | 建立半成品配方 | ✅ | ⚠️ | ⚠️ | ❌ | 创建配方时无法添加材料项；成本计算返回 0 |
| A5 | 建立菜单商品 | ✅ | ⚠️ | ⚠️ | ❌ | 创建时不传 recipe_id/category_id；无法绑定配方 |
| A6 | 设定出餐工作站 | ✅ | ⚠️ | ❌ | ❌ | 仅 seed 4 个站；无 CRUD API/UI；无法映射菜单项到工作站 |
| A7 | 设定打印/标签模板 | ✅ | ❌ | ❌ | ❌ | 完全未实现 |
| A8 | 开始营运 | ✅ | ⚠️ | ⚠️ | ⚠️ 部分 | 核心链路可跑通但体验断裂 |

**路径 A 结论：❌ 不可达**。管理者无法通过 UI 完成从 0 到 1 的完整设定。

### 路径 B：采购 → 入库 → 库存（完整链路验证）

| 步骤 | 功能 | DB | API | UI | 可达？ | 阻断点 |
|------|------|----|-----|----|--------|--------|
| B1 | 建立进货单 | ✅ | ❌ | ❌ | ❌ | 采购单表存在但无 API/UI |
| B2 | 验收入库 | ✅ | ❌ | ❌ | ❌ | 无收货 API，无法创建批次 |
| B3 | 生成批次 | ✅ | ❌ | ❌ | ❌ | 无 create_batch API |
| B4 | 贴标签 | ✅ | ❌ | ❌ | ❌ | print_tasks 表存在但无 API |
| B5 | 更新库存 | ✅ | ✅ | ✅ | ✅ | 交易流水可查看 |
| B6 | 制作半成品 | ✅ | ❌ | ❌ | ❌ | production_orders 表存在但无 API/UI |

**路径 B 结论：❌ 完全不可达**。系统没有任何方式向库存中添加原材料。

### 路径 C：POS 接单 → 提交 → 厨房 → 出餐（完整链路验证）

| 步骤 | 功能 | DB | API | UI | 可达？ | 阻断点 |
|------|------|----|-----|----|--------|--------|
| C1 | 开启 POS | ✅ | ✅ | ✅ | ✅ | 可创建订单 |
| C2 | 选择桌号/外带 | ✅ | ✅ | ❌ | ⚠️ | 后端支持 dine_type，POS UI 无选择 |
| C3 | 点餐 | ✅ | ✅ | ✅ | ⚠️ | POS 页面完整，但商品需预置 |
| C4 | 送出订单 | ✅ | ✅ | ✅ | ⚠️ | POS 只创建不提交，需跳转订单页 |
| C5 | 收款 | ❌ | ❌ | ❌ | ❌ | payments 表未建 |
| C6 | 系统自动拆单/KDS/预扣 | ✅ | ✅ | ✅ | ⚠️ | 依赖 station_menu_items 映射（无 API） |

**路径 C 结论：⚠️ 部分可达**。核心链路可跑通但体验断裂，且必须有预置数据。

### 路径 D：厨房接单 → 制作 → 出餐 → 扣料（完整链路验证）

| 步骤 | 功能 | DB | API | UI | 可达？ | 阻断点 |
|------|------|----|-----|----|--------|--------|
| D1 | 收到订单 | ✅ | ✅ | ✅ | ✅ | get_station_tickets |
| D2 | 开始制作 | ✅ | ✅ | ✅ | ✅ | start_ticket |
| D3 | 完成出餐 | ✅ | ✅ | ✅ | ✅ | finish_ticket → confirm_inventory |
| D4 | 标记完成 | ✅ | ✅ | ✅ | ✅ | 自动检查订单完成 |
| D5 | 系统自动扣料 | ✅ | ✅ | ✅ | ✅ | 预扣→实扣→释放 |

**路径 D 结论：✅ 可达**。这是系统最完整的链路。但 KDS 不显示具体菜品和备注。

## 四、阻断性问题（Critical）

| # | 问题 | 影响范围 | 修复优先级 |
|---|------|----------|-----------|
| C1 | **无创建库存批次 API** | 整个库存系统无法使用 | P0 |
| C2 | **配方成本计算返回 0** | 成本/毛利全部错误 | P0 |
| C3 | **菜单商品无法绑定配方** | 订单提交无法展开扣料 | P0 |
| C4 | **工作站-菜单映射无 API** | 拆单始终走默认站 | P1 |
| C5 | **KDS 不显示菜品和备注** | 厨师不知道做什么 | P1 |
| C6 | **POS 不自动提交订单** | 需跳转页面二次操作 | P1 |
| C7 | **所有实体无 update/delete** | 数据错误无法修正 | P1 |
| C8 | **无数据库事务** | 多步操作可能不一致 | P1 |

## 六、下一轮开发计划

### Sprint 1：打通核心数据流（P0）

1. 创建库存批次 API + UI（进货入库）
2. 修复配方成本计算（JOIN 批次获取实际成本）
3. 菜单商品绑定配方 UI
4. 所有实体 update/delete API

### Sprint 2：完善门店运营（P1）

5. 工作站-菜单映射 API + UI
6. KDS 显示菜品明细和备注
7. POS 一键提交（创建+提交合并）
8. 数据库事务支持

### Sprint 3：采购与生产（P2）

9. 采购单 CRUD + 收货入库
10. 生产单 CRUD + 执行
11. 库存调整 + 损耗记录

### Sprint 4：报表与增强（P3）

12. 销售报表
13. 毛利报表
14. 原料消耗报表
15. 热销商品

### 暂不实现（Phase 4+）

- 盘点功能
